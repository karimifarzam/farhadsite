Current problem
formai.us

Smart Contracts Lack...

The ability to determine if parts of an agreement have been fulfilled

Transactions using smart contracts are limited to the immutable properties of the blockchain, without being able to check if parts of an agreement have been fulfilled, smart contract based transaction services can never replace traditional financial services like service payments and escrow…

Oracles that interact with the real world

Current oracles are limited by their reliance on narrow, inflexible data sources, which constrains their ability to provide comprehensive and useful information for decision-making

The Solution
formai.us

Our Solution...

AI Intermediaries for transactions
AI Agents that securely interpret and execute the intent underlying an agreement/transaction
AI Arbiters capable of supervising a transaction, collecting data, and deciding whether to unlock funds
AI data collector oracles that interact with the real world and are capable of providing dynamic, context-aware, and diverse data inputs.

What is FormAI?
formai.us

FormAl is the AI infrastructure for blockchain. Our network provides the compute and code for developers to easily create and host powerful decentralized AI applications. The FormAI network is powered by decentralized physical infrastructure composed of compute nodes. The network’s Al agents connect to smart contracts and act as third party intermediaries between parties to a transaction. The FormAI network is the execution layer for AI on-chain and enables users to securely & automatically ensure the intent underlying an agreement has been fulfilled before proceeding with a transaction. We are also committed to actively supporting the ecosystem by developing in-house applications to demonstrate the utility and capabilities of FormAI.

Form Network
formai.us
Decentralized Compute Nodes

The network hosts models and processes computational workflows on decentralized physical infrastructure (DePIN) composed of compute nodes. Users meeting the minimum technical requirements can host decentralized compute nodes to support the network and receive incentives for their contribution.

FRM Token

Form Token (FRM) acts as the incentive mechanism and utility token for the FormAI network. Compute node operators are incentivized with Form token. FRM can be used alongside other cryptocurrencies to pay for services and network transaction fees.

FormAI Chain SDK

Developer friendly open source SDK to easily bring advanced AI on chain and create powerful decentralized apps. Offering libraries in Solidity & Rust to support development across blockchains (Ethereum, Solana, etc).

Governance

We are committed to implementing democratic governance mechanisms and to creating a strong and informed community. The community will be involved in decision making, which will be facilitated by the agents on the network itself, the exact mechanisms of which will be revealed with the release of the governance token.

FormAI Agents

FormAI agents are created to act as the framework for delivering practical on-chain utility using the state of the art in Machine Learning and Agentic research. Our agentic framework is modular, which allows agents to integrate new capabilities and continuously evolve their functionality.

Principles

We are prioritizing innovation, decentralization, and transparency. Our mission is to form the world’s post AGI transaction infrastructure, and the development of FormAI will reflect this fundamental goal.

import { FormAIClient } from "@formai/solidity-lib@1.0.0/FormAI.sol";

contract FreelanceMarketplace {
    FormAIClient public formAI; // Instance of the FormAI client
    ...

    enum JobState { Created, Accepted, Completed, FundsReleased }
    
    struct Job {
        address buyer;
        address seller;
        uint256 price;
        string jobDescription;
        JobState state;
        string deliverablesHash; // IPFS hash of the deliverables
    }

    mapping(uint256 => Job) public jobs;
    uint256 public jobCount;

    event JobCreated(uint256 jobId, address buyer, uint256 price, string jobDescription);
    event JobAccepted(uint256 jobId, address seller);
    event JobCompleted(uint256 jobId, string deliverablesHash);
    event FundsReleased(uint256 jobId, string explanation);

    constructor(address _formAIAddress) {
        formAI = FormAIClient(_formAIAddress);
        ...
    }

    function createJob(string memory _jobDescription) external payable {
        require(msg.value > 0, "Job price must be greater than zero.");
        
        jobs[jobCount] = Job({
            buyer: msg.sender,
            seller: address(0),
            price: msg.value,
            jobDescription: _jobDescription,
            state: JobState.Created,
            deliverablesHash: ""
        });

        emit JobCreated(jobCount, msg.sender, msg.value, _jobDescription);
        jobCount++;
    }

    function acceptJob(uint256 _jobId) external inState(_jobId, JobState.Created) {
        jobs[_jobId].seller = msg.sender;
        jobs[_jobId].state = JobState.Accepted;
        
        emit JobAccepted(_jobId, msg.sender);
    }

    function completeJob(uint256 _jobId, string memory _deliverablesHash) external onlySeller(_jobId) inState(_jobId, JobState.Accepted) {
        jobs[_jobId].deliverablesHash = _deliverablesHash;
        jobs[_jobId].state = JobState.Completed;
        
        emit JobCompleted(_jobId, _deliverablesHash);
    }

    function releaseFunds(uint256 _jobId) external onlyBuyer(_jobId) inState(_jobId, JobState.Completed) {
        // Establish connection to the oracle network
        formAI.connect();

        // Create context object
        string memory context = string(abi.encodePacked(
            "{",
            '"buyer": "', toString(jobs[_jobId].buyer), '", ',
            '"seller": "', toString(jobs[_jobId].seller), '", ',
            '"price": "', toString(jobs[_jobId].price), '", ',
            '"jobDescription": "', jobs[_jobId].jobDescription, '", ',
            '"deliverablesHash": "', jobs[_jobId].deliverablesHash, '"',
            "}"
        ));

        // Request AI inference
        (bool continueTransaction, string memory explanation) = formAI.requestInference("agent-id-456", context);

        emit FundsReleased(_jobId, explanation);

        // Check AI response and release funds if conditions are met
        require(continueTransaction, "AI check failed: Funds cannot be released.");

        payable(jobs[_jobId].seller).transfer(jobs[_jobId].price); // Transfer funds to seller
        jobs[_jobId].state = JobState.FundsReleased;
    }
}

contract GovernanceDAO {
    FormAIClient public formAI; // Instance of the FormAI client
    address public daoOwner;
    uint256 public proposalCount;

    struct Proposal {
        uint256 id;
        string description;
        uint256 votesFor;
        uint256 votesAgainst;
        bool executed;
        address proposer;
    }

    mapping(uint256 => Proposal) public proposals;
    mapping(address => bool) public members;
    mapping(address => mapping(uint256 => bool)) public votes;

    event ProposalCreated(uint256 id, string description, address proposer);
    event Voted(uint256 proposalId, address voter, bool support);
    event ProposalExecuted(uint256 proposalId, bool result, string explanation);

    constructor(address _formAIAddress) {
        formAI = FormAIClient(_formAIAddress);
        ...
    }

    function addMember(address _member) external onlyOwner {
        members[_member] = true;
    }

    function removeMember(address _member) external onlyOwner {
        members[_member] = false;
    }

    function createProposal(string memory _description) external onlyMember {
        proposalCount++;
        proposals[proposalCount] = Proposal({
            id: proposalCount,
            description: _description,
            votesFor: 0,
            votesAgainst: 0,
            executed: false,
            proposer: msg.sender
        });

        emit ProposalCreated(proposalCount, _description, msg.sender);
    }

    function vote(uint256 _proposalId, bool _support) external onlyMember {
        Proposal storage proposal = proposals[_proposalId];
        require(!proposal.executed, "Proposal already executed.");
        require(!proposal.voters[msg.sender], "Member has already voted.");

        // Establish connection to the oracle network
        formAI.connect();

        // Create context object for the AI agent to verify the vote
        string memory context = string(abi.encodePacked(
            "{",
            '"voter": "', toString(msg.sender), '", ',
            '"proposalId": "', toString(_proposalId), '", ',
            '"description": "', proposal.description, '", ',
            '"support": "', _support ? "true" : "false", '"',
            "}"
        ));

        // Request AI inference for verifying the vote
        (bool verified, string memory explanation) = formAI.requestInference("vote-verifier-agent-id", context);

        emit Voted(_proposalId, msg.sender, _support, verified, explanation);

        require(verified, "AI check failed: Vote not verified.");

        if (_support) {
            proposal.votesFor++;
        } else {
            proposal.votesAgainst++;
        }

        proposal.voters[msg.sender] = true;
    }

    function executeProposal(uint256 _proposalId) external onlyMember {
        Proposal storage proposal = proposals[_proposalId];
        require(!proposal.executed, "Proposal already executed.");

        // Establish connection to the oracle network
        formAI.connect();

        // Create context object for the AI agent to judge the proposal
        string memory context = string(abi.encodePacked(
            "{",
            '"proposalId": "', toString(_proposalId), '", ',
            '"description": "', proposal.description, '", ',
            '"votesFor": "', toString(proposal.votesFor), '", ',
            '"votesAgainst": "', toString(proposal.votesAgainst), '", ',
            '"proposer": "', toString(proposal.proposer), '"',
            "}"
        ));

        // Request AI inference for judging the proposal
        (bool result, string memory explanation) = formAI.requestInference("proposal-judge-agent-id", context);

        emit ProposalExecuted(_proposalId, result, explanation);

        proposal.executed = true;

        if (result) {
            // If AI judge decides in favor, take appropriate action (implement the proposal)
            // Placeholder for proposal implementation logic
        }
    }
}

Competitive Landscape
formai.us

Crypto x AI projects are focused on two main verticals, providing compute and providing software. Compute projects are generally either DePIN projects aggregating GPU resources (Render, Akash) or providing data services (Grass, Arkham). Projects focused on these services are competing only on price with established GPU providers like AWS/Azure and curated data providers like ScaleAI. Software focused Crypto x AI projects largely attempt to “bring AI on a chain” like ChainGPT, ICP, and BitTensor, but are competing against incumbents like OpenAI and Anthropic. FormAI leverages decentralized compute and focuses on improving transactions on the blockchain, therefore solving real problems and offering real utility while not being in competition with Web2 companies.
